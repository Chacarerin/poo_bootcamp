Este repositorio actúa como un catálogo de ejercicios y proyectos enfocados en la asimilación y aplicación del paradigma de Programación Orientada a Objetos (POO) utilizando Python. A través de una serie de sesiones de *drilling* y *rebound*, se consolida la transición desde el pensamiento procedimental hacia un enfoque basado en entidades y mensajes.

## 🧠 Contexto Pedagógico y Teórico
La Programación Orientada a Objetos es fundamental en la ingeniería de software moderna porque permite modelar problemas del mundo real a través de abstracciones (clases y objetos). Este conjunto de ejercicios está estructurado para enseñar los cuatro pilares fundamentales de la POO:
1. **Encapsulamiento:** Ocultamiento del estado interno y exposición de una interfaz pública.
2. **Abstracción:** Reducción de la complejidad ocultando detalles de implementación.
3. **Herencia:** Reutilización de código y establecimiento de jerarquías lógicas.
4. **Polimorfismo:** Capacidad de procesar objetos de diferentes clases a través de una interfaz común.

## 📁 Estructura del Repositorio
El repositorio contiene módulos independientes, cada uno asociado a una sesión académica específica:
* **Ejercicios Rebound (`sesionXrebound.py`)**: Ejercicios de calentamiento y asimilación teórica inicial.
* **Ejercicios Drilling (`sesionXdrilling.py`)**: Desafíos prácticos de mayor complejidad que requieren la integración de múltiples conceptos de diseño de software.

## ⚙️ Tecnologías y Metodologías Aplicadas
* **Python (Vanilla)**: Se utiliza Python sin frameworks externos para garantizar una comprensión pura y profunda de cómo el lenguaje maneja el modelo de objetos en memoria, el ciclo de vida de las instancias y los *dunder methods* (métodos mágicos).
* **Diseño Iterativo**: El código evoluciona desde clases simples hasta jerarquías complejas, fomentando el principio *DRY (Don't Repeat Yourself)*.
